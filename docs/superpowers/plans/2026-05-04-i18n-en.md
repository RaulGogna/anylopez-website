# AnyLopez i18n EN Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a complete English version of the site at `/en/` with auto-detection of device language and a manual 🌐 dropdown switcher in the header.

**Architecture:** Duplicate-page approach — `src/en/*.njk` pages extend the shared `base.njk` layout. Translations live in `src/_data/en/tratamientos.json`. All multilingual strings in `base.njk` are gated on a `lang` variable set by `src/en/en.json` (Eleventy directory data). Auto-detect is a small inline JS snippet using `navigator.language` + `localStorage`.

**Tech Stack:** Eleventy 11ty v3, Nunjucks, GitHub Pages, pathPrefix `/anylopez-website/`

- [ ] **Step 2.1: Update `<html>` lang attribute**

- [ ] **Step 2.2: Add hreflang URL helpers and tags in `<head>`**

After the `<meta name="robots" ...>` line, add:

```njk
  {%- set _esUrl = page.url | replace('/en/', '/') -%}
  {%- set _enUrl = '/en' + (page.url | replace('/en/', '/')) -%}
  {%- set _langPages = ['/', '/about/', '/services/', '/radiofrecuencia/', '/contact/'] -%}
  {%- if _esUrl in _langPages %}
  <link rel="alternate" hreflang="es"        href="{{ site.url }}{{ _esUrl }}" />
  <link rel="alternate" hreflang="en-GB"     href="{{ site.url }}{{ _enUrl }}" />
  <link rel="alternate" hreflang="x-default" href="{{ site.url }}{{ _esUrl }}" />
  {%- endif %}
```

**How this works:**
- `_esUrl`: strips `/en/` prefix to get the ES version of any URL. For `/about/` it returns `/about/` unchanged. For `/en/about/` it returns `/about/`.
- `_enUrl`: prepends `/en` to the ES URL. For `/` it gives `/en/`. For `/about/` it gives `/en/about/`.
- Only pages in `_langPages` get hreflang — legal pages are excluded automatically.
- `site.url = "https://raulgogna.github.io/anylopez-website"` (no trailing slash, from `_data/site.json`) — concatenated with the page URL gives the correct full canonical URL. Do NOT use the `| url` filter here (it would double the pathPrefix).
